# Contributing to Device Segmentation Pipeline

Thank you for your interest in contributing! 

## Important: Legal & Licensing

This project uses a **Dual Licensing** model (AGPLv3 for open source, Commercial License for proprietary use). 

To maintain the ability to offer commercial licenses, **we require all contributors to sign a Contributor License Agreement (CLA)** before their code can be merged. 

By submitting a Pull Request, you acknowledge that:
1. You own the copyright to your contribution.
2. You grant the project owner an irrevocable, perpetual, worldwide right to use, modify, and relicense your contribution under any license terms (including commercial).

## How to Contribute

1.  **Fork** the repository.
2.  Create a **Feature Branch** (`git checkout -b feature/AmazingFeature`).
3.  Commit your changes (`git commit -m 'Add some AmazingFeature'`).
4.  Push to the branch (`git push origin feature/AmazingFeature`).
5.  Open a **Pull Request**.

## Code Standards

* **Python:** We follow PEP-8 guidelines.
* **Environment:** Please test your code using the `Dockerfile` provided to ensure compatibility.
* **Logic:** Do not alter the core segmentation math (Loss functions, resizing logic) without explicit approval from the Algorithm Engineering team.
